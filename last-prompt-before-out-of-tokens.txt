Plan: STATMAN v1.0 — Analytics Dashboard MVP
Context
El usuario quiere iniciar STATMAN, el módulo de estadísticas/analytics de DIGI. La necesidad inmediata es analytics operacional: cuántos productos se cargaron en cuánto tiempo, precios actualizados, etiquetas impresas, distribución por categoría/estantería/género/artista/formato/país, etc.

No se necesita un app Django complejo — todos los datos ya existen en ProductV2, ImportSession, PrintJob, Venta, OrdenTrabajo. STATMAN v1 es una capa de agregación read-only sobre modelos existentes.

Hay un get_catalog_statistics endpoint existente en INVMAN que ya hace agregación MongoDB (by_category, by_type). STATMAN lo amplía significativamente.

Backend: Nuevo app statman (sin modelos, solo vistas de agregación)
Archivos a crear
backend/statman/__init__.py — vacío
backend/statman/apps.py — StatmanConfig
backend/statman/urls.py — rutas
backend/statman/views.py — Endpoint único con MongoDB aggregation pipelines
Archivos a modificar
backend/digi/settings/base.py → Agregar 'statman' a INSTALLED_APPS (línea ~41)
backend/digi/urls.py → Agregar path('api/statman/', include('statman.urls')) (línea ~33)
Endpoint: GET /api/statman/dashboard/
Un solo endpoint que retorna todo el dashboard en una llamada. Usa MongoDB $facet aggregation para eficiencia (una query, múltiples resultados).

Secciones del response:

inventory — Totales y distribuciones de ProductV2

total_products, by_category, by_type
by_genre (top 20, from data.genre)
by_artist (top 20, from data.artist)
by_country (all, from data.country)
by_format (all, from data.format)
by_location (all, from data.location)
price_stats: avg/min/max USD + cash, total inventory value
labels_printed / labels_not_printed counts
products_by_month: timeline de creación (from created_at)
imports — De ImportSession

total_sessions, total_products_imported
by_month (timeline)
recent_sessions (últimas 5)
avg_quality_score
qr_labels — De PrintJob collection

total_print_jobs, total_labels_printed
by_month (timeline)
sales — De Venta (si hay datos)

total_ventas, total_revenue
by_payment_status, by_tipo_venta
ventas_by_month
services — De OrdenTrabajo + Presupuesto (si hay datos)

total_ordenes, by_estado
total_presupuestos, presupuestos_by_estado
Modelos usados (imports en views.py):

invman.models.ProductV2 — inventory stats
invman.models.ImportSession — import stats
posman.models.Venta — sales stats
diman.models.OrdenTrabajo, Presupuesto — service stats
PrintJob: acceso directo a MongoDB collection via get_db()
